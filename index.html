Stage 01

File Pickup

Supplier SharePoint Integration

SYP files are automatically detected and retrieved from designated supplier SharePoint sites. Each supplier uploads their weekly SYP file to a dedicated folder, and our automation monitors these locations for new uploads every Thursday.

Monitors 10+ supplier SharePoint sites (Jabil Vietnam, Jabil Michigan, etc.)
Detects new SYP file uploads via Power Automate triggers
Validates file naming conventions before pickup (e.g., SYP_SupplierName_MMDDYYYY.xlsx)
Logs pickup timestamp and source metadata for audit trail
Sends automated acknowledgment to supplier on successful pickup
Process Active
Stage 01 → 02
Stage 02

Flow Processing

Power Automate Orchestration

Once picked up, the Power Automate flow orchestrates the entire processing pipeline — parsing the file, extracting key columns, applying transformations, and preparing the data for validation. The flow handles errors gracefully with retry logic and failure alerts.

Power Automate cloud flow triggers on file creation in SharePoint
Extracts and parses Excel data using Office Scripts integration
Applies column mapping and data type transformations
Handles multi-sheet workbooks and dynamic column detection
Retry logic with exponential backoff for transient failures
Process Active
Stage 02 → 03
Stage 03

Validation

AI-Powered Data Quality Checks

Every SYP file goes through a comprehensive validation engine that checks for data integrity, completeness, and consistency. The validation agent compares current submissions against prior-day data to flag anomalies, missing fields, and format violations.

Checklist validations: required columns, data types, date formats
Cross-reference with prior-day SYP data to detect discrepancies
CRD (Customer Request Date) vs Forecast shortage analysis
Copilot Studio agent highlights validation failures with detailed reasons
Alert system notifies supply planners of critical validation issues
Process Active
Stage 03 → 04
Stage 04

Snowflake Upload

Cloud Data Warehouse Ingestion

Validated and cleansed SYP data is staged and loaded into Snowflake tables. The ingestion process uses structured staging areas, merge/upsert patterns, and maintains full version history for traceability and rollback capabilities.

Data staged in Snowflake internal stage before final load
Merge logic ensures no duplicate records across daily uploads
Version history maintained with load timestamps and batch IDs
Automated Snowflake task triggers post-load transformations
Dataset refresh triggers Power BI dashboard updates downstream
Process Active
Stage 04 → 05
Stage 05

Dashboards & Alerts

Downstream Analytics & Monitoring

Once data lands in Snowflake, downstream Power BI dashboards are refreshed automatically. Supply planners and procurement teams get real-time visibility into supplier yield plans, backlog, and potential shortages — with automated email alerts on refresh failures.

Power BI dashboards refresh via scheduled triggers post-Snowflake load
Shipment & Backlog Dashboard, Universal Dashboard, Validation Dashboard
Power Automate alerts dataset owners on refresh failures
Weekly SYP summary emails sent to stakeholders automatically
Audit trail dashboard tracks file lifecycle from pickup to load
Process Active
